## 📄 요구사항 정의서 (Requirement Specification)

## 1. 개요

* 시스템명: 로컬 sLLM 기반 프라이버시 보호형 스마트홈 제어 허브
* 개발 목적: 외부 클라우드 의존성을 없애고 100% 온디바이스(On-Device) 환경에서 사용자의 자연어 문자 명령을 분석하여 스마트홈 가전제품을 안전하고 유연하게 제어하는 시뮬레이션 시스템 구축.

## 3. 기능 요구사항 정의 (Functional Requirements)

| 요구사항 ID | 기능명 | 상세 요구사항 (기능 설명) | 중요도 |
|---|---|---|---|
| FR-01 | 사용자 문자 수신 UI | 사용자가 가전 제어를 위한 자유로운 형태의 한국어 자연어 문장을 입력할 수 있는 웹 기반 텍스트 입력창과 전송 버튼을 제공해야 한다. | 필수 |
| FR-02 | 다중 명령 전처리 | 사용자가 입력한 복합 문장(예: ~하고, ~그리고 등)을 sLLM이 처리하기 쉽도록 독립된 단일 명령 단위 문장으로 분할할 수 있어야 한다. | 필수 |
| FR-03 | 자연어-JSON 변환 | 분할된 단일 자연어 문장을 로컬 sLLM에 전달하여 기기(device), 행위(action), 설정값(value)을 가진 정형화된 JSON 객체로 실시간 파싱해야 한다. | 필수 |
| FR-04 | 유연한 어휘 맵핑 | 모델이 출력한 가전 제어 어휘(예: 밝히기, 청소해줘 등)를 하드웨어가 이해할 수 있는 정형 명령 시스템 표준 어휘(예: 켜기, 끄기)로 매핑하고 예외 처리를 해야 한다. | 필수 |
| FR-05 | 실시간 가전 상태 반영 | 변환 완료된 최종 명령어를 기반으로 가상 스마트홈 가전(에어컨, 청소기, 조명, 도어락)의 상태를 UI 우측 모니터링 컴포넌트에 실시간으로 변경해 보여주어야 한다. | 필수 |
| FR-06 | 시스템 초기화 | 사용자가 가상 가전제품의 상태를 언제든지 초기 상태(꺼짐/잠김)로 리셋할 수 있는 초기화 버튼 기능을 지원해야 한다. | 보통 |

## 4. 비기능 요구사항 정의 (Non-Functional Requirements)

* 보안성 (Privacy): 가전 제어를 위한 사용자의 프라이버시 데이터 및 개인 메시지는 외부 클라우드 API를 거치지 않고 오직 로컬 환경 내부에서만 처리되어야 한다.
* 안정성 (Exception Handling): sLLM 서버 연결 실패, 잘못된 JSON 출력 등 예외 상황 발생 시 프로그램이 멈추지 않고 사용자에게 직관적인 에러 메시지(st.error)를 띄워야 한다.
* 성능 (Performance): 사용자의 복합 문장을 쪼개어 동기(Sync) 루프로 순차 처리하더라도 단일 문장당 추론 시간은 소형 모델 특성에 맞추어 최대 20초 이내(Timeout)에 완료되어야 한다.
